Gmail & Outlook: Kirim Email Otomatis

Email otomatis = follow-up lead, notifikasi order, laporan harian. Node Gmail dan Microsoft Outlook di n8n cara pakainya hampir identik — kuasai satu, kamu bisa keduanya.

Anatomi node Send Email

  • To — biasanya dinamis: {{ $json.Email }}
  • Subject — bisa campur teks + expression
  • Message — teks biasa atau HTML
  • Credentials — OAuth2 (login akun Google/Microsoft sekali)

◊ N8N CONTEXT
Setelah Code Node ini, node Gmail tinggal diisi: To = {{ $json.to }}, Subject = {{ $json.subject }}, Message = {{ $json.body }}. Karena ada 2 lead = 2 items, node Gmail otomatis jalan 2 kali — satu email per item. Ini perilaku default n8n: node dieksekusi per item.
⚠ ANTI-SPAM
Untuk volume besar, jangan kirim ratusan email sekaligus dari akun pribadi — akun bisa dibatasi Google/Microsoft. Praktik aman: batasi jumlah per eksekusi, beri jeda (node Wait), dan selalu sediakan konteks kenapa penerima mendapat email itu.

Slack & Telegram: Notifikasi & Bot

Email itu untuk pelanggan; Slack/Telegram itu untuk tim. "Ada lead baru!", "Server error!", "Order masuk!" — notifikasi real-time adalah salah satu automation paling cepat terasa nilainya bagi klien.

SlackTelegram

Format teksMarkdown ala Slack: *tebal*Markdown/HTML: <b>tebal</b>
Cocok untukTim kantor / klien korporatPersonal, UKM, klien Indonesia

Bukan cuma notifikasi: Telegram sebagai TRIGGER

Node Telegram Trigger membuat workflow berjalan saat seseorang mengirim pesan ke bot-mu. Ini webhook (F-01 Sesi 3) yang dikelola n8n secara otomatis — dasar untuk membuat bot interaktif.

[Telegram Trigger] --> [Code: parse perintah] --> [IF: perintah valid?] |-- true --> [proses] --> [Telegram: balas hasil] |-- false --> [Telegram: balas "perintah tidak dikenal"]

Step 5 / 8 · Materi 4

Airtable: Database Automation

Airtable = spreadsheet yang berperilaku seperti database. Kalau Google Sheets mulai "kewalahan" (relasi antar tabel, tipe field, view berbeda per tim), klien biasanya naik kelas ke Airtable — dan kamu harus siap.

Hirarki Airtable

Workspace '-- Base (mis. "CRM Toko") --> seperti 1 file database '-- Table (mis. "Leads", "Orders") --> seperti sheet/tab '-- Record (1 baris data) --> punya ID unik: recXXXXXXXX '-- Field (kolom bertipe: text, email, select, date...)

Dua hal yang membedakannya dari Sheets dan membuatnya terasa seperti "API sungguhan" (F-01):

  • Setiap record punya ID unik (recXXXX) — update tidak perlu "kolom pencocok", cukup ID
  • Pencarian pakai filterByFormula — query langsung di sisi server
CONTOH filterByFormula DI NODE AIRTABLE (operasi Search)read-only
// Cari semua lead berstatus "baru" dari sumber "website":
AND({Status} = "baru", {Sumber} = "website")

// Email mengandung domain tertentu:
FIND("@toko.id", {Email}) > 0

// Dibuat dalam 7 hari terakhir:
IS_AFTER({Tanggal}, DATEADD(TODAY(), -7, "days"))

Operasi node Airtable di n8n

OperasiFungsiPadanan Sheets
Search / ListAmbil records (bisa + filterByFormula)Get Row(s)
CreateTambah record baruAppend Row
UpdateUbah record via record IDUpdate Row
UpsertUpdate jika ada, create jika belum — anti duplikat(tidak ada)
◊ N8N CONTEXT
Credential Airtable memakai Personal Access Token (PAT) — dibuat di airtable.com/create/tokens dengan scope data.records:read dan data.records:write. Ini contoh nyata prinsip F-01: beri token izin seminimal yang dibutuhkan. Output node Airtable selalu menyertakan id record — simpan itu kalau nanti mau Update.
◊ QUIZ CEPAT

Error Handling & Fallback Workflow

Workflow yang jalan di laptopmu itu latihan. Workflow yang tetap selamat saat API down jam 3 pagi — itu yang dibayar klien. Ini materi yang memisahkan amatir dari profesional.

3 lapis pertahanan di n8n

LAPIS 1Pengaturan di level node — buka tab Settings di node mana pun:

  • Retry On Fail — coba ulang otomatis (mis. 3x, jeda 5 detik). Obat untuk error sementara: timeout, rate limit 429 (ingat F-01 Sesi 3)
  • On Error → Continue — workflow tidak berhenti; item gagal tetap mengalir dan bisa diarahkan ke jalur error

LAPIS 2try/catch di Code Node — skill F-02 Sesi 3-mu, langsung terpakai:

CODE NODE · proses tiap item, tandai yang gagal (jangan crash)editable
Output akan muncul di sini... (perhatikan: 1 dari 3 item sengaja dibuat error)

LAPIS 3Error Workflow — workflow terpisah dengan node Error Trigger. Di Settings workflow utama, pilih dia sebagai "Error Workflow". Saat eksekusi mana pun gagal total, dia otomatis berjalan membawa info error: nama workflow, node yang gagal, dan pesan errornya.

WORKFLOW UTAMA (gagal total) ........ memicu .......> ERROR WORKFLOW [Error Trigger] | [Slack/Telegram] "Workflow X gagal di node Y: pesan Z"

Pola fallback: jalur cadangan

Fallback = rencana B di dalam workflow. Contoh: jika Slack gagal mengirim, kirim lewat Telegram; jika keduanya gagal, catat ke sheet "failed_notifications".

[Slack] --(On Error: Continue)--> [IF: ada field error?] |-- false --> selesai (sukses) |-- true --> [Telegram] --> [Sheets: log kegagalan]
◊ N8N CONTEXT
Saat node di-set On Error: Continue (using error output), node mendapat output kedua berwarna merah khusus untuk item gagal — jalur fallback dirangkai dari output itu. Inilah implementasi visual dari konsep try/catch.
◊ QUIZ CEPAT
API klien sering melempar error 429 (rate limit) selama beberapa detik lalu normal lagi. Pertahanan PALING tepat untuk kasus ini?

Mission: Lead Management System

Project resmi sesi ini dari roadmap: Form → Sheet → Email → Slack notif. Arsitektur lengkapnya:

[Webhook/Form Trigger] | [Code: validasi + format] <-- yang kamu kerjakan di mission ini | [Google Sheets: Append] | [Gmail: email sambutan ke lead] | [Code: ringkasan] --> [Slack: notif tim #leads]
◊ MISSION BRIEF

Tulis Code Node "validasi + format" dengan aturan:

  • Buang lead yang emailnya tidak mengandung "@" (pakai .filter())
  • Setiap lead valid menghasilkan field: Nama (trim), Email (lowercase), Sumber (default "website" jika kosong)
  • Return array of items format n8n: { json: {...} }

Data uji berisi 4 lead: 1 email rusak, 1 tanpa sumber, 1 nama berspasi berlebih. Output benar = 3 item bersih.
