STEP 2 / 8

Instalasi n8n: Cloud vs Self-Hosted

Ada dua jalur menjalankan n8n. Sebagai calon automation expert, kamu harus paham keduanya — klien akan bertanya.

n8n CLOUDSELF-HOSTED
SetupDaftar di n8n.io → langsung jalan (kamu sudah di sini)Install sendiri via Docker / npm di server (Railway, Render, VPS)
BiayaLangganan bulanan per eksekusi/workflowSoftware gratis (license fair-code), bayar server saja
DataLewat server n8n100% di infrastruktur sendiri — penting untuk klien dengan data sensitif
MaintenanceDiurus n8n (update, uptime)Kamu yang urus update, backup, monitoring
Webhook URLOtomatis tersedia & publicPerlu domain / tunnel agar bisa diakses publik
◈ KONTEKS BISNIS

Rute belajar: mulai di Cloud (fokus ke skill workflow, bukan server). Rute klien nanti: tawarkan self-hosted di Railway/Render saat klien butuh kontrol data atau volume eksekusi tinggi — ini ada di daftar tools fase F-03.

Perintah self-hosted (cukup kenali dulu)

# Opsi 1: npm (butuh Node.js — sudah kamu kenal dari F-02)
npx n8n

# Opsi 2: Docker (cara paling umum di produksi)
docker run -it --rm -p 5678:5678 docker.n8n.io/n8nio/n8n

Belum perlu dipraktikkan sekarang. Deployment produksi dibahas di Sesi 3 (Advanced n8n).

STEP 3 / 8

Interface & Konsep Dasar Workflow

Buka editor n8n-mu. Yang kamu lihat adalah canvas — papan tempat menyusun workflow.

  • Workflow = rangkaian langkah otomatis, dari trigger sampai output
  • Node = satu kotak = satu tugas (ambil data, ubah data, kirim data)
  • Connection = garis antar node = jalur aliran data
  • Items = data yang mengalir: selalu berupa array of JSON object
  • Execution = satu kali workflow berjalan dari awal sampai akhir

4 Node Utama: HTTP Request, Set, IF, Merge

Dengan 4 node ini saja kamu sudah bisa membangun mayoritas automation sederhana.

1 — HTTP Request

Postman-nya n8n. Method, URL, headers, body, authentication — semua istilah F-01 Sesi 2 dipakai persis di sini. Output-nya: response API sebagai items.

⚡ LIVE: HTTP REQUEST KE JSONPLACEHOLDER
Klik tombol untuk melihat response sebagai item n8n...

2 — Set (Edit Fields)

Node transformasi: pilih, ganti nama, atau buat field baru. Untuk mengambil nilai dari item masuk, Set memakai referensi {{ $json.namaField }} — cukup kenali pola dasar ini dulu; pendalaman expressions ada di Sesi 3 (Advanced n8n).

3 — IF

Percabangan: item dicek terhadap kondisi, lalu keluar lewat output true atau false. Ini if/else dari F-02 dalam bentuk visual.

STEP 5 / 8

Trigger: Manual, Schedule, Webhook

Setiap workflow dimulai dari trigger — node pertama yang menentukan KAPAN workflow berjalan.

TRIGGERKAPAN JALANCONTOH KASUS
ManualSaat kamu klik tombol ExecuteTesting & development workflow
ScheduleOtomatis sesuai jadwal (tiap jam, tiap hari 08:00, dst.)Laporan harian, sinkronisasi data rutin
WebhookSaat sistem luar mengirim HTTP request ke URL n8n-muForm submission, notifikasi payment gateway
◈ KONEKSI F-01 SESI 3

Masih ingat trigger-based vs polling? Webhook trigger = trigger-based (sistem luar yang mendorong data, real-time). Schedule trigger + HTTP Request = pola polling (kamu yang rutin menjemput data). Sekarang teorinya jadi node nyata.

◈ PENTING: TEST URL vs PRODUCTION URL

Webhook node punya dua URL. Test URL hanya aktif saat kamu klik "Listen for test event" di editor. Production URL baru aktif setelah workflow di-set Active. Ini sumber kebingungan #1 pemula n8n.

STEP 6 / 8

Debug & Execution Log

Skill yang membedakan amatir dan profesional: bukan menulis workflow tanpa error, tapi cepat menemukan di mana error terjadi.

  • Klik node setelah eksekusi → lihat input & output data per node
  • Node merah = node tempat error terjadi; baca pesan errornya, jangan ditebak
  • Menu Executions = riwayat semua eksekusi (sukses/gagal) beserta datanya
  • Pin data: kunci output sebuah node agar testing node berikutnya tidak perlu memanggil API berulang kali (hemat rate limit — F-01 Sesi 3)
  • Status eksekusi mengikuti pola yang sudah kamu tahu: error API umumnya soal status code 4xx/5xx

STEP 7 / 8

Project Praktik: 3 Workflow Dasar

Sesuai roadmap: data fetch → transform → output. Kerjakan langsung di akun n8n Cloud-mu. Pakai API latihan dari F-01 supaya fokus ke n8n, bukan ke API baru.

WORKFLOW A — POLA DASAR
User Fetcher
Manual Trigger→ HTTP Request→ Set

GET https://jsonplaceholder.typicode.com/users → di Set node, buat output rapi: hanya field nama, email, kota (kota ada di dalam address.city — nested object, F-01!).

WORKFLOW B — POLLING + PERCABANGAN
Kurs Watcher
Schedule Trigger→ HTTP Request→ IF→ Set

Schedule tiap 1 jam → GET https://api.frankfurter.dev/v1/latest?base=USD&symbols=IDR → IF: kurs IDR lebih besar dari ambang pilihanmu → branch true: Set membuat field status: "ALERT", branch false: status: "NORMAL". Test dengan Execute manual dulu.

WORKFLOW C — WEBHOOK + MERGE
Order Combiner
Webhook→ Set (data order)→ Merge

Webhook menerima POST berisi JSON order (kirim test pakai Postman/Hoppscotch — tools F-01). Cabang kedua: HTTP Request mengambil data user JSONPlaceholder. Merge mode Combine menjodohkan keduanya lewat field id. Coba juga mode Append untuk merasakan bedanya.
