📌 Kenapa JS untuk Automation?

Di F-01 kamu belajar JSON — format data yang digunakan semua API. Sekarang kamu butuh cara untuk memanipulasi data itu: filter, transform, gabungkan, kondisikan.


JavaScript adalah bahasa yang digunakan di n8n Code Node — node paling powerful di n8n. Ketika built-in nodes tidak cukup, Code Node bisa melakukan apa saja.

◈ N8N CODE NODE — INI YANG AKAN KAMU TULIS

Ini contoh nyata kode di n8n Code Node:

n8n Code Node — contoh real
// Data masuk dari HTTP Request node sebelumnya
const items = $input.all();

// Proses setiap item
const result = items.map(item => {
  const nama = item.json.customer.name;
  const total = item.json.order.total;
  const isPremium = total > 500000;

  return {
    nama,
    total,
    tier: isPremium ? "Premium" : "Standard",
    pesan: `Halo ${nama}, order Rp ${total.toLocaleString('id-ID')} berhasil!`
  };
});

return result;

Setelah sesi ini, kamu sudah paham setiap baris kode di atas.

📚 Yang Akan Dipelajari Hari Ini

  • Variables: let dan const (skip var)
  • Data types: string, number, boolean, null
  • Objects — JSON yang bisa dimodifikasi
  • Arrays — daftar data (sangat umum di API)
  • Template literals — string dinamis
  • Conditionals: if/else dan ternary
⚠️ Disclaimer penting: Tujuanmu BUKAN jadi programmer JS. Tujuanmu adalah bisa menulis logic di n8n Code Node. Kita skip banyak hal yang tidak relevan untuk automation.
💡 Tips belajar: Setiap topik punya code editor interaktif. Jangan cuma baca — ketik sendiri, ubah-ubah, lihat hasilnya. Begitu cara tercepat JS masuk ke otak.
STEP 01 · F-02 SESI 1
Variables: let & const
Variables adalah cara menyimpan data dengan nama. Di JS modern, kita pakai dua kata kunci: const dan let.

CONST — nilai yang tidak berubah

Gunakan const sebagai default. Sekali diberi nilai, tidak bisa di-reassign.

const — constant value
const apiKey = "sk-abc123xyz";
const baseUrl = "https://api.example.com";
const maxRetries = 3;
const isProduction = true;

// ❌ Ini akan ERROR — const tidak bisa di-reassign
// apiKey = "sk-baru"; // TypeError!

LET — nilai yang mungkin berubah

Gunakan let hanya ketika nilainya perlu berubah di kemudian kode.

let — mutable value
let totalProcessed = 0;
let currentStatus = "pending";
let retryCount = 0;

// ✅ let bisa diubah
totalProcessed = totalProcessed + 1;
currentStatus = "success";
retryCount++;

console.log(totalProcessed); // 1
console.log(currentStatus);  // "success"
❌ Skip var sepenuhnya. var adalah cara lama yang punya masalah scope. Di automation modern, tidak ada alasan pakai var.
💡 Rules of thumb: Mulai selalu dengan const. Kalau nanti kamu butuh ubah nilainya, ganti ke let. Ini best practice industry.

STEP 03 · F-02 SESI 1
Objects — JSON yang Hidup
Di F-01 kamu baca JSON sebagai teks. Sekarang kamu akan membuat, mengubah, dan memanipulasi object langsung dari kode.
membuat & akses object
// Buat object
const customer = {
  id: "cust_001",
  nama: "Reza Pratama",
  email: "reza@example.com",
  umur: 28,
  isPremium: true,
};

// Akses dot notation (umum, lebih bersih)
console.log(customer.nama);      // "Reza Pratama"
console.log(customer.email);     // "reza@example.com"
console.log(customer.isPremium); // true

// Akses bracket notation (untuk dynamic key)
const field = "umur";
console.log(customer[field]);    // 28

// Modifikasi nilai
customer.umur = 29;
customer.kota = "Jakarta";  // tambah properti baru
console.log(customer.umur); // 29
console.log(customer.kota); // "Jakarta"

// Hapus properti
delete customer.kota;
console.log(customer.kota); // undefined

🔗 NESTED OBJECTS — sangat umum di API

API response sering punya object di dalam object. Pakai chaining untuk akses:

nested objects — API response typical
const apiResponse = {
  status: "success",
  code: 200,
  data: {
    order: {
      id: "ord_789",
      total: 450000,
      customer: {
        nama: "Reza",
        email: "reza@example.com"
      }
    }
  }
};

// Akses nested — pakai dot notation berantai
console.log(apiResponse.status);                        // "success"
console.log(apiResponse.data.order.id);                 // "ord_789"
console.log(apiResponse.data.order.customer.nama);      // "Reza"
console.log(apiResponse.data.order.customer.email);     // "reza@example.com"
💡 Pattern JSON F-01 → JS Object: JSON yang kamu pelajari di F-01 adalah representasi teks dari JS object. Saat JSON di-parse dengan JSON.parse(), langsung jadi JS object yang bisa kamu manipulasi.

Arrays — Daftar Data
Array adalah daftar berurutan. Sangat umum di automation — list email, list order, list customer, semua datang sebagai array.
arrays — dasar
// Buat array
const emails  = ["reza@mail.com", "budi@mail.com", "ani@mail.com"];
const angka   = [100, 200, 300, 400];
const mixed   = ["teks", 42, true, null];  // boleh mix tipe

// Akses by index — mulai dari 0!
console.log(emails[0]); // "reza@mail.com"  ← index 0
console.log(emails[1]); // "budi@mail.com"  ← index 1
console.log(emails[2]); // "ani@mail.com"   ← index 2
console.log(emails[3]); // undefined         ← tidak ada

// Jumlah elemen
console.log(emails.length); // 3

// Elemen terakhir (trick umum)
const last = emails[emails.length - 1];

// Tambah dan hapus
emails.push("deni@mail.com");    // tambah di akhir
console.log(emails.length);      // 4
emails.pop();                    // hapus dari akhir
console.log(emails.length);      // 3 lagi

STEP 05 · F-02 SESI 1
Template Literals
Cara modern untuk membangun string dinamis. Sangat berguna untuk membuat pesan, email body, subject, dan notifikasi di automation workflow.
template literals — syntax dasar
const nama   = "Reza";
const tier   = "Premium";
const saldo  = 1500000;

// ❌ Cara lama — string concatenation (repot, rawan typo)
const msg1 = "Halo " + nama + "! Akun " + tier + " kamu aktif. Saldo: Rp " + saldo;

// ✅ Template literal — pakai BACKTICK (`) bukan quote!
const msg2 = `Halo ${nama}! Akun ${tier} kamu aktif. Saldo: Rp ${saldo}`;

console.log(msg1);
console.log(msg2);
// Output sama, tapi template literal jauh lebih bersih
💡 Backtick `` ` `` bukan quote! Template literal pakai karakter backtick (` `) — di keyboard biasanya di pojok kiri atas, sebelah tombol 1. Variable/ekspresi disisipkan dengan ${...}.
multi-line & expressions
const customer = { nama: "Budi", email: "budi@mail.com" };
const order    = { id: "ORD-001", total: 450000, items: 3 };

// Multi-line (sangat berguna untuk email body)
const emailBody = `
Halo ${customer.nama},

Order kamu berhasil diproses!
Order ID  : ${order.id}
Total     : Rp ${order.total.toLocaleString('id-ID')}
Jumlah    : ${order.items} item

Terima kasih sudah belanja!
`.trim();

console.log(emailBody);

// Expression di dalam ${}
const diskon = order.total * 0.1;
const afterDiskon = `Total setelah diskon: Rp ${order.total - diskon}`;
console.log(afterDiskon);

// Kondisi (ternary) di dalam template
const badge = `Status: ${order.total > 300000 ? "🏆 VIP" : "👤 Regular"}`;
console.log(badge);

STEP 06 · F-02 SESI 1
Conditionals
Logic percabangan — dasar dari semua decision-making di automation. Apakah order perlu di-approve? Customer tier apa? Status sukses atau gagal?
if / else / else if
const orderTotal = 750000;
const stockLevel = 3;

// if / else dasar
if (orderTotal > 500000) {
  console.log("Free ongkir!"); // jalan karena 750000 > 500000
} else {
  console.log("Ongkir dikenakan");
}

// else if untuk multiple kondisi
const status = "failed";
if (status === "paid") {
  console.log("✅ Proses pengiriman");
} else if (status === "pending") {
  console.log("⏳ Tunggu pembayaran");
} else if (status === "failed") {
  console.log("❌ Pembayaran gagal — kirim notif");
} else {
  console.log("❓ Status tidak dikenal");
}

// Comparing — gunakan === (strict), bukan == (loose)
console.log(1 === 1);    // true
console.log(1 === "1");  // false ← berbeda tipe!
console.log(1 == "1");   // true  ← loose (HINDARI)
ternary operator — shortcut if/else
const amount  = 850000;
const isPaid  = true;
const tier    = "premium";

// if/else biasa
let label;
if (amount > 500000) { label = "🏆 VIP"; } else { label = "👤 Regular"; }

// ternary — lebih singkat untuk kondisi sederhana
const label2  = amount > 500000 ? "🏆 VIP" : "👤 Regular";
const status  = isPaid ? "Lunas" : "Belum bayar";
const route   = tier === "premium" ? "#vip-channel" : "#general";

console.log(label2);   // "🏆 VIP"
console.log(status);   // "Lunas"
console.log(route);    // "#vip-channel"

// Ternary di dalam template literal (powerful combo!)
const msg = `Customer ${label2} — ${isPaid ? "✅" : "⚠️"} — Route: ${route}`;
truthy & falsy — penting untuk automation
// Nilai FALSY di JS (evaluasi jadi false di kondisi):
// false, 0, "", null, undefined, NaN

// Nilai TRUTHY: semua yang lain

const email = null;
const nama  = "Reza";
const stok  = 0;

// Cek dengan if
if (email) {
  console.log("Email ada:", email);
} else {
  console.log("Email kosong!"); // ini yang jalan (null = falsy)
}

if (nama) {
  console.log("Nama:", nama); // ini yang jalan ("Reza" = truthy)
}

if (stok) {
  console.log("Ada stok");
} else {
  console.log("Stok habis!"); // ini yang jalan (0 = falsy)
}
◈ N8N CODE NODE — CONDITIONAL LOGIC

Di n8n, conditional dalam Code Node menggantikan IF node untuk logic yang lebih kompleks:

n8n — routing dan filtering
// Contoh: proses order dengan routing logic
const items = $input.all();

return items.map(item => {
  const data = item.json;
  
  // Tentukan priority
  const priority = data.amount > 1000000 ? "HIGH" : 
                   data.amount > 300000  ? "MEDIUM" : "LOW";
  
  // Tentukan channel notifikasi
  const channel = data.status === "failed"  ? "#alerts" :
                  priority === "HIGH"       ? "#vip-orders" : "#general";
  
  // Buat message
  const msg = `[${priority}] Order ${data.id} — ${data.customer} — ${channel}`;
  
  return { json: { ...data, priority, channel, msg } };
});
