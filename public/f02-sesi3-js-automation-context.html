STEP 2 OF 8 · MATERI 01
Async/Await Dasar
JavaScript berjalan satu jalur (single-threaded). Async/await memungkinkan kita "menunggu" operasi lambat seperti network request tanpa membekukan seluruh program.
◆ KONSEP INTI
TIGA KATA KUNCI
›
Promise — representasi nilai yang akan ada di masa depan (resolve atau reject)
›
async — tandai function sebagai "bisa melakukan operasi async"
›
await — "pause di sini sampai Promise selesai, baru lanjut ke baris berikutnya"
⚠ Aturan wajib: await hanya bisa digunakan di dalam function yang ditandai async. Tanpa itu, JavaScript throw SyntaxError.
◆ SYNTAX
// Cara lama: Promise chain fetch(url) .then(res => res.json()) .then(data => console.log(data)) .catch(err => console.log(err)); // Cara baru: async/await — lebih bersih async function ambilData() { const res = await fetch(url); const data = await res.json(); console.log(data); }
◆ DI N8N CODE NODE
n8n Code Node berjalan secara async. Kamu bisa langsung pakai await di top-level tanpa wrapper function extra.
// Langsung di Code Node: const res = await fetch('https://api.example.com/data'); const data = await res.json(); return [{ json: data }];

STEP 3 OF 8 · MATERI 02
fetch() untuk HTTP Request
fetch() adalah fungsi bawaan JavaScript untuk membuat HTTP request. Mengembalikan Promise — selalu gunakan dengan async/await.
◆ ANATOMY FETCH
async function ambilData() { // Step 1: kirim request, tunggu response const response = await fetch('https://api.example.com/data'); // Step 2: parse body sebagai JSON (juga async!) const data = await response.json(); console.log(data); }
⚠ Dua kali await: satu untuk fetch() (dapat header), satu untuk response.json() (baca body). Lupa await kedua = dapat Promise, bukan data!
◆ CEK RESPONSE STATUS
const response = await fetch(url); // response.ok = true jika status 200-299 if (!response.ok) { throw new Error("HTTP Error: " + response.status); } const data = await response.json();
◆ REFERENSI CEPAT
PROPERTYCONTOHKEGUNAAN
response.oktrue / falsecek request berhasil
response.status200, 404, 500kode status HTTP
await response.json()Object / Arrayparse body sebagai JSON
await response.text()Stringbaca body sebagai teks
◆ DI N8N CODE NODE
fetch() bekerja sama di n8n. Untuk request sederhana gunakan HTTP Request Node saja. Code Node + fetch() digunakan saat butuh logic khusus sebelum atau sesudah request.

STEP 4 OF 8 · MATERI 03
Error Handling: try/catch
API bisa down. Jaringan bisa putus. Data bisa tidak sesuai format. try/catch memastikan workflow tetap berjalan dengan baik meski ada yang salah.
◆ STRUKTUR
try { // kode yang MUNGKIN gagal const data = await ambilDariAPI(); console.log(data); } catch (error) { // dijalankan JIKA ada error di try console.log("Error:", error.message); } finally { // SELALU dijalankan, ada error atau tidak console.log("Selesai"); }
FUNGSI SETIAP BLOK
try
Kode berjalan normal. Jika error → langsung lompat ke catch. Baris setelah error tidak dieksekusi.
catch
Menerima object error berisi error.message. Bisa log, kirim notifikasi, atau set default value.
finally
Selalu jalan — untuk cleanup: set loading = false, tutup koneksi, log timestamp selesai.
◆ DI N8N CODE NODE
Error yang tidak di-handle akan stop workflow. Dengan try/catch kamu bisa: log error ke Slack/Sheet, kirim ke error workflow, atau return data fallback — inilah yang membuat workflow production-ready.

STEP 5 OF 8 · MATERI 04
Date & Time Manipulation
Timestamp, format tanggal, menghitung waktu — semua butuh Date object. Di automation, date dipakai untuk logging, scheduling, dan memberi konteks pada data.
◆ MEMBUAT DATE OBJECT
const sekarang = new Date(); // waktu saat ini const spesifik = new Date('2024-06-15'); // dari string ISO const timestamp = Date.now(); // ms sejak epoch, tanpa new
◆ METHOD PENTING
METHODCONTOH HASILCATATAN
getDate()1 – 31hari dalam bulan
getMonth()0 – 110 = Januari! Harus +1
getFullYear()2024tahun 4 digit
getHours()0 – 23format 24 jam
getMinutes()0 – 59menit
toISOString()2024-06-15T08:30:00.000Zformat standar API
Date.now()1718438400000milliseconds sejak epoch
⚠ Gotcha klasik: getMonth() mengembalikan 0 untuk Januari, 11 untuk Desember. Selalu tambah +1 saat menampilkan ke user atau menyimpan ke database!
◆ FORMAT CUSTOM DD/MM/YYYY
const now = new Date(); const dd = String(now.getDate()).padStart(2, '0'); const mm = String(now.getMonth() + 1).padStart(2, '0'); const yyyy = now.getFullYear(); const fmt = dd + "/" + mm + "/" + yyyy; // "15/06/2024"
◆ DI N8N CODE NODE
Timestamp sering dipakai untuk: (1) audit trail — catat kapan data diproses, (2) file naming — laporan-2024-06-15.pdf, (3) scheduling — apakah sudah lewat deadline?

STEP 6 OF 8 · MATERI 05
Regex Dasar untuk Parsing Data
Regex adalah pola untuk mencari, memvalidasi, dan mengekstrak teks. Di automation: parsing log, validasi input, membersihkan data sebelum diproses.
◆ SYNTAX DASAR
// Format: /pola/flags const p1 = /hello/i; // i = case-insensitive const p2 = /\d+/g; // g = global (cari semua match)
◆ TIGA METHOD UTAMA
const teks = "Error 404 di 2024-06-15"; // test() — apakah pola ada? → true/false /error/i.test(teks); // true // match() — ambil yang cocok → array atau null teks.match(/\d+/g); // ["404","2024","06","15"] teks.match(/\d{4}-\d{2}-\d{2}/); // ["2024-06-15"] // replace() — ganti yang cocok → string baru teks.replace(/\d+/g, "X"); // "Error X di X-X-X"
◆ POLA YANG SERING DIPAKAI
POLAARTICONTOH MATCH
\dsatu digit 0-93, 7
\d+satu atau lebih digit42, 1000
\d{4}tepat 4 digit2024
\w+kata (huruf/angka/_)hello, user_1
\S+bukan whitespaceemail@test.com
flag icase-insensitiveERROR = error
flag gcari semua matchtanpa g = hanya pertama
💡 Tip aman: Gunakan teks.match(/pola/)?.[0] — jika tidak ada match (null), optional chaining ?.[0] tidak throw error, hanya return undefined.

STEP 7 OF 8 · INTEGRASI
Semua Bersatu
Inilah pattern yang ditulis di n8n Code Node: async/await + fetch + try/catch + date + regex — semua bersama membangun automation yang solid.
◆ PATTERN LENGKAP
// N8N CODE NODE: Fetch API -> Process -> Format Output async function processData() { const timestamp = new Date().toISOString(); // DATE try { // TRY/CATCH // 1. FETCH const res = await fetch('https://api.example.com'); // ASYNC/AWAIT if (!res.ok) throw new Error("API Error: " + res.status); const data = await res.json(); // FETCH // 2. PROCESS + REGEX const hasil = data.map(function(item) { const words = (item.title.match(/\w{4,}/g) || []); // REGEX return { id: item.id, keywords: words, processedAt: timestamp }; }); // 3. RETURN KE WORKFLOW N8N return hasil.map(item => ({ json: item })); } catch (err) { console.log("Error:", err.message); return [{ json: { error: err.message, timestamp: timestamp } }]; } }
ALUR DATA
1
INPUT — Data dari node sebelumnya atau fetch dari API
2
PROCESS — Filter, transform, extract dengan JS (map, filter, regex, date)
3
OUTPUT — Return array [{ json: ... }] untuk node berikutnya
