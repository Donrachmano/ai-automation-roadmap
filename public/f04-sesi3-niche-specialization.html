Kenapa Niche = Differentiator

Sesi 1 & 2 mengajarkan kamu LLM API dan agentic workflow. Sekarang pertanyaannya: untuk siapa? Niche adalah jawaban yang membuat skill itu laku mahal.

Generalist vs Specialist

"Saya bisa automation apa saja" terdengar fleksibel, tapi di mata klien itu sinyal lemah — tidak ada yang langsung kepikiran memanggil kamu untuk masalah spesifik mereka. Bandingkan:

  • Generalist: "AI automation freelancer" → bersaing harga dengan ribuan orang.
  • Specialist: "Saya bangun lead-scoring + follow-up otomatis untuk agensi real estate" → klien langsung paham nilainya, harga jadi soal ROI bukan jam kerja.

Memilih Niche Vertikal

Jangan pilih niche karena "kelihatan keren". Pilih dengan kerangka keputusan yang bisa diukur.

4 Kriteria Penilaian

  • Demand — apakah banyak bisnis punya masalah ini sekarang?
  • Pain jelas — apakah sakitnya nyata & berulang (bukan sekali setahun)?
  • Budget — apakah niche ini punya uang untuk membayar solusi?
  • Domain fit — apakah kamu cukup paham konteksnya untuk terdengar kredibel?

Beri skor 1–5 tiap kriteria, lalu jumlahkan. Niche dengan total tertinggi adalah taruhan terbaik untuk dimulai. Code Node di bawah meniru cara n8n memproses array — $input.all() jadi data, kamu transform dengan .map() & .sort().

// STEP 03

Bangun Workflow Template

Template bagus = config-driven. Logika tetap, yang berubah cuma parameter. Jadi satu workflow bisa dipakai ulang lintas klien.

Prinsip Reusable Template

  • Parameterisasi — ambang skor, label, email tujuan disimpan sebagai config, bukan di-hardcode di tengah node.
  • Normalisasi input — data dari form/CRM apa pun dirapikan ke satu bentuk standar dulu.
  • Idempoten & aman — pakai item?.json dan default value agar tidak meledak saat field kosong.

Contoh template niche CRM: normalisasi lead lalu beri skor. Perhatikan optional chaining & default — pola wajib di n8n karena data klien sering tidak rapi.

// Code Node: normalisasi + scoring lead (CRM niche)
var config = { hot: 70, warm: 40 };

return $input.all().map(function (item) {
  var lead = item?.json || {};
  var email = (lead.email || "").trim().toLowerCase();
  var score = 0;
  if (email.indexOf("@") > 0) score += 30;
  if (lead.company) score += 25;
  if (Number(lead.budget) > 1000) score += 25;
  var tier = score >= config.hot ? "HOT"
           : score >= config.warm ? "WARM" : "COLD";
  return { json: { email: email, score: score, tier: tier } };
});
◆ KONTEKS n8n
Setelah Code Node ini, sambung ke node IF (cek tier === "HOT") → cabang HOT kirim Slack alert ke sales, cabang lain masuk Google Sheet untuk nurture. Itulah anatomi template yang siap dijual.

Coba sendiri: ubah ambang config atau tambah sinyal skor, lalu RUN.

Pricing & Packaging

Jangan jual jam. Jual hasil. Paket yang dikemas rapi membuat klien memilih "yang mana", bukan "iya atau tidak".

Tiga Model Harga

  • Project-based — harga tetap per workflow. Cocok untuk klien pertama & scope jelas.
  • Retainer — biaya bulanan untuk maintenance + perubahan. Income paling stabil.
  • Productized — paket standar siap-jual (misal "CRM Starter") — skalanya paling tinggi.

Tiga Tier (Good / Better / Best)

STARTER

$500
  • 1 workflow inti
  • Setup + handover
  • 7 hari support

GROWTH

$1.500
  • 3 workflow
  • Error handling
  • 30 hari support
  • 1x revisi

SCALE

$500/bln
  • Retainer
  • Monitoring
  • Perubahan rutin
  • Prioritas
◆ VALUE-BASED THINKING
Kalau automation kamu menghemat 20 jam/bulan staf seharga $15/jam, itu $300/bulan nilai — sebelum menghitung lead yang tertangkap lebih cepat. Harga $1.500 jadi terlihat murah saat dibingkai dari ROI, bukan dari effort kamu.

Dokumentasi Case Study

Case study adalah bukti. Tanpa angka, itu cuma cerita. Struktur yang menjual: Context → Problem → Solution → Result.

Anatomi Case Study

  • Context — siapa kliennya & industrinya (1–2 kalimat).
  • Problem — sakit yang spesifik & terukur ("balas lead butuh 6 jam").
  • Solution — workflow yang kamu bangun, ringkas.
  • Result — angka before/after. Ini bagian yang dibaca duluan.

Code Node di bawah mengubah objek case study menjadi ringkasan markdown — persis pola yang kamu pakai untuk auto-generate dokumentasi portfolio. Regex yang dipakai sederhana & ASCII (aman dari backtracking).

// STEP 06

Portfolio Presentation

Klien tidak membaca kode kamu. Mereka butuh melihat hasil dalam 60 detik. Presentasi yang jelas mengubah skill jadi kepercayaan.

Tiga Format Wajib

  • Loom demo (1–2 menit) — tunjukkan workflow jalan dari trigger ke output. Visual > deskripsi.
  • Workflow-as-code di GitHub — export JSON n8n + README. Bukti kamu rapi & bisa di-handover.
  • One-pager solusi — problem, solusi, hasil, harga — dalam satu halaman scannable.

Bingkai dengan STAR

Saat presentasi tiap proyek, ceritakan dengan: Situation → Task → Action → Result. Kerangka ini memaksa kamu menutup dengan dampak, bukan dengan daftar fitur.

◆ JEMBATAN KE F-05
Paket portfolio yang kamu rapikan di sini langsung jadi amunisi F-05 (Market Entry): profil Upwork, proposal, dan discovery call semuanya menarik dari case study + one-pager + demo Loom yang kamu siapkan sekarang.
⚠ HINDARI
Portfolio penuh jargon teknis tanpa konteks bisnis. "Saya pakai HTTP Request node + Merge + Code Node" tidak berarti apa-apa untuk pemilik bisnis. Terjemahkan ke bahasa hasil: "lead masuk → tim ditugaskan otomatis dalam 2 menit".
